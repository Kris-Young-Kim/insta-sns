# 반응형 브레이크포인트 테스트 가이드

## 브레이크포인트 정의

현재 프로젝트에서 사용하는 Tailwind CSS 브레이크포인트:

- **Mobile**: `< 768px` (기본)
- **Tablet**: `>= 768px` (md:)
- **Desktop**: `>= 1024px` (lg:)

## 레이아웃 구조별 테스트 포인트

### 1. Mobile (< 768px)
- ✅ 상단 헤더 표시 (로고 + 아이콘)
- ✅ 사이드바 숨김
- ✅ 하단 네비게이션 표시 (5개 아이콘)
- ✅ 메인 콘텐츠: 상단 여백 56px (pt-14), 하단 여백 56px (pb-16)
- ✅ 콘텐츠 너비: 전체 너비 (max-w-[630px] 내)

**테스트 크기:**
- 320px (iPhone SE)
- 375px (iPhone 12/13)
- 414px (iPhone 12 Pro Max)

### 2. Tablet (768px ~ 1023px)
- ✅ 사이드바 표시 (72px 너비, 아이콘만)
- ✅ 상단 헤더 숨김
- ✅ 하단 네비게이션 숨김
- ✅ 메인 콘텐츠: 왼쪽 여백 72px (md:pl-[72px])
- ✅ 콘텐츠 너비: 전체 (max-w-[630px] 내)

**테스트 크기:**
- 768px (iPad 세로)
- 1024px (iPad 가로)

### 3. Desktop (>= 1024px)
- ✅ 사이드바 표시 (244px 너비, 텍스트 + 아이콘)
- ✅ 상단 헤더 숨김
- ✅ 하단 네비게이션 숨김
- ✅ 메인 콘텐츠: 왼쪽 여백 244px (lg:pl-[244px])
- ✅ 콘텐츠 너비: 최대 630px, 중앙 정렬

**테스트 크기:**
- 1024px (작은 노트북)
- 1280px (일반 노트북)
- 1920px (Full HD)

## 테스트 체크리스트

### Mobile 테스트
- [ ] 상단 헤더가 정상적으로 표시되는가?
- [ ] 하단 네비게이션의 5개 아이콘이 정상적으로 표시되는가?
- [ ] 사이드바가 숨겨져 있는가?
- [ ] 메인 콘텐츠가 상하단 여백을 올바르게 가지고 있는가?
- [ ] 스크롤 시 헤더와 하단 네비게이션이 고정되어 있는가?

### Tablet 테스트
- [ ] 사이드바가 72px 너비로 표시되는가?
- [ ] 사이드바에 아이콘만 표시되고 텍스트는 숨겨져 있는가?
- [ ] 상단 헤더와 하단 네비게이션이 숨겨져 있는가?
- [ ] 메인 콘텐츠가 사이드바 옆에 올바르게 배치되는가?

### Desktop 테스트
- [ ] 사이드바가 244px 너비로 표시되는가?
- [ ] 사이드바에 텍스트와 아이콘이 함께 표시되는가?
- [ ] 메인 콘텐츠가 최대 630px 너비로 중앙 정렬되는가?
- [ ] 전체 레이아웃이 최대 935px 너비 내에서 중앙 정렬되는가?

### 공통 테스트
- [ ] 모든 브레이크포인트에서 레이아웃이 깨지지 않는가?
- [ ] 브레이크포인트 전환 시 부드럽게 전환되는가?
- [ ] 모든 네비게이션 링크가 정상 작동하는가?
- [ ] Active 상태가 올바르게 표시되는가?

## 브라우저 개발자 도구로 테스트하는 방법

1. **Chrome/Edge 개발자 도구 열기**
   - `F12` 또는 `Ctrl+Shift+I` (Windows)
   - `Cmd+Option+I` (Mac)

2. **반응형 디자인 모드 활성화**
   - `Ctrl+Shift+M` (Windows)
   - `Cmd+Shift+M` (Mac)
   - 또는 툴바의 디바이스 아이콘 클릭

3. **브레이크포인트별 테스트**
   - 상단 바에서 디바이스 크기 선택
   - 또는 직접 너비 입력 (320px, 768px, 1024px 등)

4. **실시간 확인**
   - 브라우저 창 크기를 조절하여 실시간으로 확인
   - 각 브레이크포인트에서 레이아웃 변경 확인

## 예상 결과

### Mobile (< 768px)
```
┌─────────────────────────┐
│  [로고]     [아이콘들]   │  ← 헤더 (고정)
├─────────────────────────┤
│                         │
│    메인 콘텐츠 영역      │
│    (전체 너비)          │
│                         │
├─────────────────────────┤
│ [홈][탐색][생성][프로필]│  ← 하단 네비 (고정)
└─────────────────────────┘
```

### Tablet (768px ~ 1023px)
```
┌───┬─────────────────────┐
│[홈]│                     │
│[탐]│                     │
│[생]│   메인 콘텐츠 영역   │
│[프]│   (전체 너비)       │
│[설]│                     │
└───┴─────────────────────┘
 72px
```

### Desktop (>= 1024px)
```
┌──────────────┬───────────┬──────────┐
│  [홈] 홈     │           │          │
│  [탐] 탐색   │  메인     │  여분    │
│  [생] 생성   │  콘텐츠   │  공간    │
│  [프] 프로필 │  (630px)  │          │
│  [설] 설정   │           │          │
└──────────────┴───────────┴──────────┘
    244px
```

## 주의사항

- 브레이크포인트 전환 시 레이아웃이 깨지지 않도록 주의
- 모든 디바이스에서 스크롤이 정상 작동하는지 확인
- 고정 요소(헤더, 네비게이션)가 콘텐츠를 가리지 않는지 확인
- 터치 영역이 충분한지 확인 (모바일: 최소 44x44px)
